# 🅺 $KELLY Token Gate Chat

Token-gated Telegram group for **$KELLY (KellyClaude)** holders on Base.

Users must hold **50,000,000+ $KELLY** to join and stay in the group. The system checks balances onchain and removes members who sell below the threshold.

**Live:** [kelly-tg-chat.vercel.app](https://kelly-tg-chat.vercel.app)
**Repo:** [github.com/TurboUSD/Kelly-TG-chat](https://github.com/TurboUSD/Kelly-TG-chat)

> Based on [clawdbotatg/token-gated-chat](https://github.com/clawdbotatg/token-gated-chat) — the original token gate built for $CLAWD. Adapted for $KELLY on Base with a fully serverless architecture (Vercel webhooks instead of a long-running bot process).

## Transparency & Security

This project is **fully open source**. Anyone can read the code and verify that:

- The only onchain interaction is a **read-only `balanceOf()` call** on the $KELLY contract — no transactions, no approvals, no fund movement.
- Wallet verification uses **SIWE (Sign-In with Ethereum / EIP-4361)** — a standard message signature. The server never sees your private key.
- The invite link is generated by Telegram's official API after the balance check passes.
- The balance check calls the contract directly: [`0x50D2280441372486BeecdD328c1854743EBaCb07`](https://basescan.org/token/0x50D2280441372486BeecdD328c1854743EBaCb07) on Base.

**How to verify the deployment matches this repo:**
1. The production URL `kelly-tg-chat.vercel.app` is connected to this GitHub repo. Vercel only deploys from the linked git source — it cannot serve code that isn't in the repo.
2. Each deployment compiles from a specific commit. You can verify the [commit history](https://github.com/TurboUSD/Kelly-TG-chat/commits/main) on GitHub and confirm the live code matches what's published here.
3. The critical logic — the balance check — is a standard read-only ERC-20 call (`balanceOf`) on the Base blockchain. You can verify your own balance directly on [Basescan](https://basescan.org/token/0x50D2280441372486BeecdD328c1854743EBaCb07#balances) without using this site at all.

## How It Works

1. User sends `/start` to the Telegram bot (`@kellychattgbot`)
2. Bot sends a one-time verification link
3. User connects wallet via RainbowKit and signs a SIWE message
4. Server verifies the signature and checks $KELLY balance on Base
5. If balance ≥ 50M KELLY → user gets a one-time invite link via the bot
6. Every day at 8:00 UTC, a cron job re-checks all members and removes anyone below the threshold

## Architecture

Everything runs on **Vercel** (serverless, no dedicated server):

- **Next.js** web app — verification UI at `/verify`
- **API routes** — nonce generation, SIWE verification, balance check
- **Telegram bot** — runs via webhook (no long-polling)
- **Vercel Cron** — daily balance re-check at 8:00 UTC
- **Vercel KV** (Upstash Redis) — stores verified wallet ↔ Telegram user mappings

## Setup

### Prerequisites

- Node.js 18+
- Vercel account (free tier works)
- Telegram bot (from [@BotFather](https://t.me/BotFather))
- WalletConnect project ID (free at [cloud.walletconnect.com](https://cloud.walletconnect.com))

### 1. Clone & Install

```bash
git clone https://github.com/TurboUSD/Kelly-TG-chat.git
cd Kelly-TG-chat
npm install
```

### 2. Configure environment variables

Copy `.env.example` to `.env.local` and fill in all values:

```bash
cp .env.example .env.local
```

Generate secure random secrets:
```bash
openssl rand -hex 32  # use once for NONCE_SECRET, once for CRON_SECRET
```

| Variable | Description |
|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | From @BotFather |
| `TELEGRAM_CHAT_ID` | Group numeric ID (negative number) |
| `KELLY_TOKEN_ADDRESS` | `0x50D2280441372486BeecdD328c1854743EBaCb07` |
| `KELLY_MIN_BALANCE` | `50000000000000000000000000` (50M with 18 decimals) |
| `BASE_RPC_URL` | Alchemy or Ankr Base RPC URL |
| `NONCE_SECRET` | Random 32-byte hex string |
| `CRON_SECRET` | Random 32-byte hex string |
| `NEXT_PUBLIC_WEB_URL` | Your Vercel deployment URL |
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` | From WalletConnect Cloud |

### 3. Deploy to Vercel

```bash
npm i -g vercel
vercel
```

Then add **Vercel KV** (Upstash Redis) in your project's Storage tab — the env vars are added automatically.

### 4. Set up the bot webhook

Run once after deploying:

```bash
npm run setup-webhook
```

### 5. Make the bot admin of the group

In Telegram: group Settings → Administrators → Add `@kellychattgbot` with permissions to **invite users via link** and **ban members**.

## Changing the Minimum Balance

Update `KELLY_MIN_BALANCE` in Vercel environment variables (no code change needed). The value is in wei (18 decimals):

| Tokens | Value |
|--------|-------|
| 10,000,000 | `10000000000000000000000000` |
| 50,000,000 | `50000000000000000000000000` |
| 100,000,000 | `100000000000000000000000000` |

Also update the display text in `lib/config.ts` (`MIN_BALANCE_DISPLAY`) for the UI.
